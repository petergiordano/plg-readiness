# Modern Web UI Design Guidelines

This document sets the rules and examples for creating modern, high-quality UI and UX for web applications.
Use it before generating HTML, CSS, components, or layout ideas.

**Applies to:** SaaS products, web apps, marketing sites, dashboards, AI-first tools, and general web interfaces.

---

## 1. Design Goals

- Modern, clean, simple.
- Strong spacing and clear hierarchy.
- Product screenshots or UI elements are the focus.
- Text is clear and short.
- Pages feel calm and confident, not loud.
- Layout supports conversion and easy reading.

---

## 2. Core Principles

- Use white space to guide the eye.
- Use large headings and clear body text.
- Use one or two accent colors.
- Avoid clutter, gradients, and random shapes.
- Structure comes first. Visuals second.
- Make each section have one clear job.
- Show the product early and often.
- Use consistent spacing (4, 8, 16, 24, 32px scale).

---

## 3. UX Tone and Feel

- Calm.
- Premium.
- Minimal.
- Confident.
- Clear and direct.

Avoid noisy visuals, tiny text, cramped layouts, or heavy decoration.

---

## 4. Required Workflow

### Step 1: Text Wireframe

Describe the page section by section. No styling. No code.

### Step 2: Semantic HTML

Write clean HTML only. No inline styles. No classes unless needed.

### Step 3: Modern CSS or Tailwind

Focus on:
- Spacing
- Typography
- Contrast
- Layout grid
- Clean cards

### Step 4: Critique Pass

Review the result like a senior product designer.

Fix:
- Generic layouts
- Poor contrast
- Weak hierarchy
- Too much noise

## 6. Layout Guidelines

- Hero should be simple with one clear message.
- Use short text + a clear button.
- Show product screens early.
- Use asymmetric layouts when possible.
- Limit sections. Remove anything not needed.
- Use card layouts for feature grids.
- Use simple container widths and consistent padding.

---

## 7. Typography Rules

- One or two fonts only.
- Large headings (32–56px).
- Comfortable body text (16–18px).
- Bold for structure, not decoration.
- Keep line width readable.

---

## 8. Color and Visual Rules

- Light background by default.
- High contrast text.
- One accent color used sparingly.
- Avoid bright gradients or neon tones.
- No random 3D objects, blobs, or floating shapes.

---

## 9. Component Style Rules

### Buttons
- Solid or outline.
- Simple shape.
- No heavy shadows.

### Cards
- Subtle border or very light shadow.
- Clear title + short text.
- Even internal spacing.

### Navigation
- Simple, clean, sticky if needed.
- Clear CTA on the right.

### Screenshots
- Clean frames.
- No fake device chrome unless needed.
- Keep them crisp and readable.

---

## 10. Common Commands to Use With AI

Use these prompts with Claude, ChatGPT, or other AI tools:

### Wireframe
> "Create a text-only wireframe for a modern [homepage/dashboard/landing page]. Keep each section clear and simple."

### HTML
> "Turn this wireframe into clean, semantic HTML with no styling."

### CSS
> "Add modern minimal CSS. Focus on spacing, typography, and contrast. No gradients or heavy decoration."

### Variation
> "Give me three layout variations that each feel different in structure and tone."

### Critique
> "Critique this design like a senior designer. Identify what feels generic. Rewrite the layout to fix it."

### Responsive
> "Make this layout responsive. Mobile-first approach. Ensure touch targets are at least 44px."

---

## 11. Quality Bar

The final product should feel:
- Modern
- Simple
- Premium
- Conversion-ready
- Easy to scan
- Easy to extend

**If the UI feels like a generic AI template, revise.**
