Rationale (per RESEARCH.md "Out of scope for this research"): adding preconnect would (a) introduce a 7th `<head>` element, (b) fire a fresh D-14 browser-verify gate (BV-5), (c) marginally improve font-fetch latency (sub-100ms, not load-bearing for any acceptance criterion). - Weight subset: stick to the current weight stops (Space Grotesk 400/500/600/700; Plus Jakarta Sans 400/500/600; JetBrains Mono 400). Per RESEARCH.md File-Level Scope Google Fonts edit: Plus Jakarta Sans 700 is not consumed by any current markup (every `font-bold` usage co-occurs with `font-display`, confirmed via re-grep at A6 in RESEARCH.md Assumptions Log). **Threat-model coverage:** T-2-03 (Google Fonts `<link>` href swap) — accept; same trust boundary as Phase 1. Space Grotesk is hosted on `fonts.gstatic.com`, same as the retired Fraunces. No new third-party domain. See `<threat_model>` below.
